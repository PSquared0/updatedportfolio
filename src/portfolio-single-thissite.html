Project Overview

The Problem

A portfolio built years ago for graphic design clients no longer reflected where I was headed. Every project was framed around deliverables — "here's the logo," "here's the campaign" — with no context for why it existed or what it solved. It also required editing the same nav, footer, and header across a dozen separate HTML files every time anything changed. The site was hard to maintain and harder to reposition.

What I Did

  • Rebuilt the entire site using a Gulp 4 partials system — nav, footer, and CTA are maintained in one file each, not duplicated across every page
  • Rewrote all case study copy using a problem → process → outcome framework to speak to product and ops hiring managers, not just creative directors
  • Designed and implemented a rotating accent palette (5 warm tones) using CSS custom properties, with all combinations WCAG AA verified for contrast
  • Added a paired rotating headline system — each palette loads a different hero copy angle reflecting a different hiring use case
  • Built animated components from scratch: morphing SVG blob hero, scroll-triggered stat counters using IntersectionObserver, smooth eased number animation via requestAnimationFrame
  • Wrote the About page experience section, career stats, and tools stack to make the ops and project management background legible to technical hiring managers

Outcome

A portfolio that reflects both design craft and technical depth — and changes its story slightly with every visit, because flexibility is the whole point. Every design decision has a reason. Every line of code is mine.
